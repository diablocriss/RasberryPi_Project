# Spec: Sneat UI Redesign

## Goal
Replace the current plain-Bootstrap hotel booking UI with a proper Sneat admin-template
layout (vertical sidebar, topbar, content area) using the local copy of the template as
reference. Vendor assets are embedded in the binary so the Pi works fully offline.

## Reference template location
```
sneat-bootstrap-html-admin-template-free-v3.0.0/
  sneat-bootstrap-html-admin-template-free/
    html/
      index.html               ← layout shell (sidebar + navbar + content)
      cards-basic.html         ← card grid patterns
      form-layouts-vertical.html ← form field layout
      tables-basic.html        ← data table patterns
    dist/
      css/core.css             ← 585 KB — only CSS file needed
      js/bootstrap.js          ← 336 KB
      js/helpers.js            ←  32 KB
      js/menu.js               ←  24 KB
```

## Design decisions

### Asset delivery: embed in binary (not CDN, not filesystem)
The Pi may have no internet. Embedding vendor CSS/JS with `xxd` is consistent with the
existing HTML-embedding pattern and adds only ~978 KB to the binary — acceptable on Pi 4.

Four new embedded headers will be generated:
```
src/vendor_core_css.h       ← from web/vendor/core.css
src/vendor_bootstrap_js.h   ← from web/vendor/bootstrap.js
src/vendor_helpers_js.h     ← from web/vendor/helpers.js
src/vendor_menu_js.h        ← from web/vendor/menu.js
```

Served on routes:
```
GET /vendor/css/core.css       → Content-Type: text/css
GET /vendor/js/bootstrap.js    → Content-Type: application/javascript
GET /vendor/js/helpers.js      → Content-Type: application/javascript
GET /vendor/js/menu.js         → Content-Type: application/javascript
```

### Fonts: keep Google Fonts CDN
Public Sans is decorative only. Pages remain fully usable offline (system fonts fall back).
No extra embedding needed.

### Only 4 vendor files — no extras
Sneat's demo also uses `perfect-scrollbar`, `apex-charts`, `iconify-icons`, `demo.css`,
and `config.js`. None of these are needed for a hotel booking app.

### HTML rewrite strategy
Both pages use the **same Sneat vertical-menu shell** from `html/index.html`:
```
div.layout-wrapper.layout-content-navbar
  aside#layout-menu           ← sidebar
  div.layout-container-navbar
    nav.layout-navbar          ← topbar
    div.content-wrapper
      div.container-xxl.flex-grow-1.container-p-y
        [page content here]
```

`web/index.html` (guest page) content area contains:
1. Date range picker + room-type filter (from existing JS logic)
2. Room availability card grid (pattern from `cards-basic.html`)
3. Booking form (pattern from `form-layouts-vertical.html`)

`web/admin.html` content area contains:
1. Bookings data table with Cancel buttons (pattern from `tables-basic.html`)
2. Refresh button in topbar or card header

### JavaScript: keep existing application logic verbatim
All `fetch()` calls, DOM IDs, event handlers, and flatpickr usage in the current
`web/index.html` and `web/admin.html` are correct and must be preserved exactly.
Only the surrounding HTML structure and CSS classes change.

## Constraints
- Never edit `src/index_html.h`, `src/admin_html.h`, or any `src/vendor_*.h` by hand —
  always regenerate with `make`.
- `src/web.c` must compile cleanly with `-Wall -Wextra`. No unused variables.
- `web.c` must not use `malloc` / heap — keep the static `header[256]` pattern.
- All existing REST API routes in `api.c` are unchanged.

## Out of scope
- Sneat demo-only extras: `perfect-scrollbar`, `apex-charts`, `iconify-icons`, `config.js`,
  `demo.css` — not needed, not embedded.
- Authentication / login page — already out of scope for the whole project.
- Favicon embedding — adds 4 KB of noise for negligible benefit.
- Dark mode / theme switcher — Sneat ships it but we are not wiring it up.
- Any changes to `api.c`, `booking.c`, `storage.c`, `server.c`, `main.c`.
