# Skill: web-ui

---

## Purpose

Design and edit the embedded web dashboard (guest booking page + admin page) and the
static vendor asset layer. Covers HTML rewrites, Sneat template usage, vendor CSS/JS
embedding, `src/web.c` route extension, and the full Makefile xxd pipeline.

---

## When to use

- Changing the layout, structure, or styling of `web/index.html` or `web/admin.html`
- Adding or changing UI controls (forms, buttons, tables) that call the REST API
- Adding a new embedded static asset (CSS, JS) served from the binary
- Extending `src/web.c` with a new HTTP route
- Debugging a page not loading or a vendor asset returning 404
- Planning any UI work that references the Sneat admin template

## Makefile xxd rules — naming convention

The `sed` command renames the xxd-generated C symbol to strip the path prefix:

| Source file | sed pattern | Resulting C symbols |
|---|---|---|
| `web/index.html` | `s/web_index_html/index_html/g` | `index_html[]`, `index_html_len` |
| `web/admin.html` | `s/web_admin_html/admin_html/g` | `admin_html[]`, `admin_html_len` |
| `web/vendor/css/core.css` | `s/web_vendor_css_core_css/vendor_core_css/g` | `vendor_core_css[]`, `vendor_core_css_len` |
| `web/vendor/js/bootstrap.js` | `s/web_vendor_js_bootstrap_js/vendor_bootstrap_js/g` | `vendor_bootstrap_js[]`, `vendor_bootstrap_js_len` |
| `web/vendor/js/helpers.js` | `s/web_vendor_js_helpers_js/vendor_helpers_js/g` | `vendor_helpers_js[]`, `vendor_helpers_js_len` |
| `web/vendor/js/menu.js` | `s/web_vendor_js_menu_js/vendor_menu_js/g` | `vendor_menu_js[]`, `vendor_menu_js_len` |

**Rule template** (copy for any new asset):
```makefile
src/<symbol>.h: web/<path/to/asset>
	xxd -i $< | sed 's/<xxd_default_symbol>/<symbol>/g' > $@
```

## web.c route dispatch pattern

`src/web.c` uses a single `web_serve(fd, path)` function. Add new routes by extending
the `if/else if` chain — one branch per path, set `html`, `html_len`, and `content_type`:

```c
void web_serve(int fd, const char *path) {
    const unsigned char *html = index_html;
    unsigned int         html_len = index_html_len;
    const char          *content_type = "text/html; charset=utf-8";
    char header[256];

    if      (strcmp(path, "/admin") == 0)                  { html = admin_html;         html_len = admin_html_len;         }
    else if (strcmp(path, "/vendor/css/core.css") == 0)    { html = vendor_core_css;    html_len = vendor_core_css_len;    content_type = "text/css"; }
    else if (strcmp(path, "/vendor/js/bootstrap.js") == 0) { html = vendor_bootstrap_js; html_len = vendor_bootstrap_js_len; content_type = "application/javascript"; }
    else if (strcmp(path, "/vendor/js/helpers.js") == 0)   { html = vendor_helpers_js;  html_len = vendor_helpers_js_len;  content_type = "application/javascript"; }
    else if (strcmp(path, "/vendor/js/menu.js") == 0)      { html = vendor_menu_js;     html_len = vendor_menu_js_len;     content_type = "application/javascript"; }

    int hlen = snprintf(header, sizeof(header),
        "HTTP/1.1 200 OK\r\nContent-Type: %s\r\nContent-Length: %u\r\nConnection: close\r\n\r\n",
        content_type, html_len);
    ssize_t r;
    r = write(fd, header, hlen); (void)r;
    r = write(fd, html, html_len); (void)r;
}
```

Also update `src/api.c` — the condition that routes to `web_serve` must include any new
path alongside `/` and `/admin`.

## Constraints and gotchas

- **Never edit any `src/*.h` header by hand** — they are all overwritten on `make`.
- `helpers.js` must be loaded in `<head>`, before `<body>`. The other JS files go at the
  end of `<body>`.
- Vendor files in `web/vendor/` are copies of the Sneat dist files — do not edit them.
  To update Sneat, re-copy from the dist folder and rebuild.
- All application JS (`fetch`, DOM IDs, event handlers, flatpickr) must be preserved
  verbatim when restructuring HTML — only surrounding markup and CSS classes change.
- Timestamps from the API are Unix seconds — convert with `new Date(ts * 1000)` in JS.
- `web.c` must not use `malloc` — keep the stack-allocated `char header[256]` pattern.
- Build with `-Wall -Wextra`: no unused variables, no implicit casts.

## Patterns

### Fetch availability
```javascript
fetch(`/api/availability?check_in=${ci}&check_out=${co}`)
  .then(r => r.json())
  .then(j => { if (!j.ok) { /* show error */ return; }
               renderRoomCards(j.data); });
```

### POST booking
```javascript
fetch('/api/bookings', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ room_id, guest_name, guest_email, check_in, check_out })
}).then(r => r.json()).then(j => { ... });
```

### Cancel booking
```javascript
fetch(`/api/bookings/${id}`, { method: 'DELETE' })
  .then(r => r.json()).then(j => { if (j.ok) loadBookings(); });
```

### Timestamp formatting
```javascript
function fmt(ts) { return new Date(ts * 1000).toLocaleDateString(); }
```

## Step-by-step: editing the dashboard

1. Edit `web/index.html` or `web/admin.html`.
2. Run `make` — triggers all xxd rules, recompiles `web.c`, relinks binary.
3. Start: `./appbooking -p 8080 -d /tmp/test.db`
4. Open `http://localhost:8080/` (or `/admin`).
5. Check DevTools Network tab: `/vendor/css/core.css` must return 200.

### Adding a new embedded asset

1. Copy the file to `web/vendor/<subpath>`.
2. Add an xxd rule to `Makefile` (see naming convention table above).
3. Add the header to the deps of `build/obj/web.o`, `build/obj-pi/web.o`, `build/obj-pi64/web.o`.
4. `#include` the new header in `src/web.c`.
5. Add an `else if` branch in `web_serve()` with the correct `Content-Type`.
6. Add the path to the `web_serve` guard in `src/api.c`.
7. Reference the new path in the HTML (`<link>` or `<script>`).
